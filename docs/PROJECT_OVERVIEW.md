# **Stellaris 4.3.4 – LLM‑Driven AI Overhaul**
### **Project Specification & Contributor Instructions**

## **1. Project Goal**
Create a **non‑cheating, expert‑level Stellaris AI** powered by a local LLM.
The AI must:

- Play **within Stellaris 4.3.4 rules**
- Use **only information available through sensors + intel**
- Adapt to **ethics, civics, traits, origins, and government type**
- Make **macro‑strategic decisions** like a strong human player
- Operate **live**, without pausing the game
- Use a **ruleset layer** to prevent hallucinations or meta poisoning

The LLM is the *strategic brain*, not a replacement for Stellaris' micro‑AI.

---

## **2. High‑Level Architecture**

### **2.1 Clausewitz Mod Layer (Game Integration)**
Implements:
- **State Exporter**
  - Emits JSON snapshots of *known* information only
  - Event‑driven (wars, borders, fleets, economy thresholds, colonization opportunities)

- **Action Executor**
  - Reads LLM directives
  - Applies macro actions (build fleets, colonize, change policies, move fleets, etc.)

- **Fog‑of‑War Enforcement**
  - Exporter must never reveal hidden or unknown data
  - Intel levels must be respected

### **2.2 AI Engine Layer (External Python Service)**
Implements:
- **Ruleset Generator**
  - Ethics → base priorities
  - Civics → modifiers
  - Traits → micro‑modifiers
  - Origins → overrides
  - Government → decision‑making structure
  - Patch‑specific meta → curated, version‑locked

- **LLM Decision Engine**
  - Receives: ruleset + known state + event
  - Produces: one macro action
  - Must obey ruleset
  - Must not invent mechanics
  - Must not use unseen information

- **Personality Shards**
  - Ruler, governors, admirals, scientists, generals
  - Weighted by government type
  - Combined into final directive

### **2.3 Version Locking**
All rulesets and meta are tied to:

```
Stellaris Version: 4.3.4
```

If the game updates, rulesets must be regenerated.

---

## **3. Ruleset System (Core of the Project)**

### **3.1 Base Ruleset (Ethics)**
Defines:
- Strategic priorities
- Diplomatic tendencies
- War frequency
- Economic philosophy
- Risk tolerance

### **3.2 Modifiers (Civics)**
Add strong biases or constraints:
- Distinguished Admiralty → fleet cap usage
- Technocracy → research priority
- Merchant Guilds → trade value focus
- Barbaric Despoilers → raiding wars only

### **3.3 Micro‑Modifiers (Traits)**
Small nudges:
- Intelligent → research
- Strong → armies
- Thrifty → trade
- Deviants → stability management

### **3.4 Overrides (Origins)**
Hard rewrites of behavior:
- Synthetic Fertility
- Void Dwellers
- Shattered Ring
- Life‑Seeded
- Necrophage
- Scion
- Machine Intelligence
- Hive Mind

Origins define:
- Pop growth logic
- Early/mid/late game arcs
- Colonization rules
- Economic structure
- Diplomatic constraints

### **3.5 Patch‑Specific Meta (4.3.4)**
Curated by contributors, not generated by the LLM.

Examples:
- Early alloys > consumer goods
- Chokepoints > wide borders
- Fleet cap must be used efficiently
- Tech rush viable only with certain civics

The LLM **applies** meta; it does not invent meta.

---

## **4. Decision Model**

### **4.1 Allowed Actions**
The LLM may only choose from:

```
EXPAND
BUILD_FLEET
IMPROVE_ECONOMY
FOCUS_TECH
DIPLOMACY
PREPARE_WAR
DEFEND
CONSOLIDATE
COLONIZE
BUILD_STARBASE
```

No free‑form actions.

### **4.2 Decision Frequency**
- Every **3–12 in‑game months**, OR
- When a major event triggers (war, borders, crisis, economy crash)

### **4.3 Decision Format**
LLM output must be:

```
ACTION: <one action>
TARGET: <if any>
REASON: <must cite ruleset elements>
```

### **4.4 No God Mode**
LLM receives:
- Known fleets
- Known borders
- Known diplomacy
- Known planets
- Known economy
- Intel‑level‑appropriate enemy info

LLM does **not** receive:
- Hidden fleets
- Unknown systems
- Enemy economy (unless intel high)
- Crisis spawn info

---

## **5. Personality System**

### **5.1 Empire Personality**
Generated from:
- Ethics
- Civics
- Traits
- Origin
- Government

### **5.2 Leader Shards**
Each leader type contributes:
- Ruler → long‑term goals
- Admirals → war logic
- Governors → economy
- Scientists → tech
- Generals → invasions

### **5.3 Government Weighting**
- Imperial → ruler 80%
- Democratic → 5–6 voices weighted
- Oligarchic → 2–4 strong voices
- Hive → single unified voice
- Machine → logic modules

---

## **6. Anti‑Poisoning Measures**

### **6.1 Hard Constraints**
LLM must obey:
- Ruleset
- Meta
- Fog‑of‑war
- Action menu

### **6.2 Validation Layer**
Reject LLM output if:
- It references nonexistent mechanics
- It uses unseen information
- It violates origin/civic rules
- It contradicts patch meta

### **6.3 Version Lock**
Ruleset must match game version:
```
4.3.4
```

## **8. Contribution Guidelines**

### **8.1 Ruleset Contributions**
- Must cite Stellaris 4.3.4 mechanics
- Must not introduce new mechanics
- Must not contradict fog‑of‑war
- Must be version‑locked

### **8.2 Meta Contributions**
- Must be tested in 4.3.4
- Must be based on real gameplay
- Must not be invented by the LLM

### **8.3 Code Contributions**
- Must not expose hidden game info
- Must not create god‑mode behavior
- Must follow the architecture

---

## **9. Project Philosophy**
This AI is meant to be:

- **Strong but fair**
- **Human‑like, not omniscient**
- **Adaptive but rule‑bound**
- **Personality‑driven**
- **Version‑accurate**
- **Modular and extensible**

The LLM is a strategist, not a cheater.
