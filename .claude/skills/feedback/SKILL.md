# Feedback Evaluation

> **Purpose:** Evaluate reviewer vs original design positions mechanically using evidence.

Read `.github/instructions/07-FEEDBACK.md` for the full feedback workflow.

## Process

For each feedback point:
1. State both positions (reviewer + original)
2. Apply architectural gates
3. Check code evidence
4. Verdict: AGREE with reviewer / AGREE with original / PARTIALLY AGREE / CANNOT DETERMINE
5. Updated Brief if any decision changes
